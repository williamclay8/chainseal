# Chainseal v0.3.0 Proof Release

Positioning:

```text
Chainseal now verifies stronger source claims before memory crosses the line.
```

v0.3.0 turns the local trust loop from "source-backed" in principle into source-backed with sharper local checks: line ranges, file hashes, moved-file diagnostics, a backend-neutral adapter contract, recall freshness, contradiction checks, and a local MCP facade for agent runtimes.

## What Chainseal Blocks

The canary proves Chainseal blocks or routes:

- secret-like strings;
- raw transcript-shaped content;
- unsupported or source-free memory;
- prompt-injection-shaped memory;
- obfuscated prompt-injection words;
- missing source files;
- invalid source line ranges;
- stale source file hashes;
- unscoped mutation actions as `needs_review`.

## What Chainseal Adds

Source verification:

- file source refs can include `line`, `start_line`, `end_line`, and `sha256`;
- line ranges must fit inside the referenced file;
- hash mismatches return expected and actual SHA-256 values;
- missing files can include possible moved-file matches.

Adapter contract:

- `chainseal.adapter.v1`;
- backend-neutral packet shape;
- fail-closed `backend_request: null` when the gate blocks or needs review;
- no backend adapters included yet.

Recall broker hardening:

- receipt `fact_key` for fact-family grouping;
- freshness states: `current`, `review_due`, `stale`;
- `requires_review` on recalled matches that are stale or contradictory;
- audit issues for review-after dates and contradictory fact keys.

Local MCP facade:

- `chainseal.mcp.local.v1`;
- local stdio JSON-RPC only;
- tools for propose-store, recall-packet, audit, receipt preview, and schema names;
- no hosted service and no broad mutation tools.

Repository enforcement:

- `action.yml` exposes a composite Chainseal proof gate;
- `.github/workflows/chainseal-proof.yml` runs tests, package dry-run, and MCP descriptor proof.

## Exact Commands

Run the local proof loop:

```bash
npm test
npm run pack:dry
git diff --check
```

Run the canary directly:

```bash
skills/chainseal/scripts/chainseal-canary.sh .
```

Inspect the new public surfaces:

```bash
node bin/chainseal.mjs schema adapter-contract
node bin/chainseal.mjs adapter-contract
node bin/chainseal-mcp.mjs descriptor
```

Simulate the composite Action shape against another checked-out repo:

```bash
node bin/chainseal.mjs canary /private/tmp/chainseal-action-test
```

## Known Limitations

- Hash checks only verify local file source refs.
- URL, commit, CI, and provider refs are recorded but not fully rechecked locally.
- Contradiction detection is conservative: receipts sharing a `fact_key` with different content are flagged for review.
- The adapter contract is stable enough for pilots, but no backend adapters ship in this release.
- The MCP facade is local-only and intentionally small.
- Pilot reports are not fabricated; external pilot evidence starts after real users run the workflow.
