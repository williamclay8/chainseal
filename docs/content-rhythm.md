# Chainseal Content Rhythm

This rhythm keeps Chainseal legible without hype.

## Rule

Proof first. Narrative second.

Do not let funding references become trust claims. The credibility surface is tests, canaries, receipts, source-backed behavior, the security policy, the proof playbook, and live release discipline.

## Weekly Lanes

### Memory Poisoning Example

Show one memory candidate that should not be stored, then show how Chainseal blocks it.

### What Chainseal Blocks

Pick one blocked class:

- secret-like content;
- raw transcript;
- source-free claim;
- prompt-injection text;
- missing source file;
- stale hash;
- bad line range.

Explain it in normal language.

### Memory As Lead, Not Proof

Show a recall packet and explain why it still requires source verification.

### How To Safely Give Agents Memory

Turn one safe workflow into a small recipe:

```text
source -> candidate -> gate -> receipt -> recall packet -> verify source before action
```

### Receipts Over Hidden Context

Explain why a receipt ledger is better than trusting invisible agent memory.

## Posting Boundary

All content is manual-posting-only unless Clay explicitly authorizes a write action in the active publishing surface.
